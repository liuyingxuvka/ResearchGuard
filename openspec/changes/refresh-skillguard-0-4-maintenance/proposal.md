## Why

ResearchGuard's four-member skill suite still carries author-maintenance artifacts generated by an older SkillGuard contract shape, while the repository itself lacks the current explicit author-adoption identity. This prevents current maintenance and validation claims even though the standalone consumer skills remain usable.

## What Changes

- Adopt this repository as the single current SkillGuard author source for the inseparable `researchguard`, `logicguard`, `sourceguard`, and `traceguard` maintenance unit.
- Regenerate each member's compiled contract and check manifest directly with SkillGuard 0.4, preserving the target-declared native route, checks, evidence subjects, and execution owners.
- Freeze model-derived structure and test boundaries so old standalone repositories, OpenSpec artifacts, author receipts, and consumer distributions cannot become alternate authorities.
- Produce a clean source-to-consumer projection comparison and current affected-validation evidence without installing or publishing the consumers.
- Keep old standalone LogicGuard, SourceGuard, and TraceGuard repositories untouched; their preservation and later retirement remain a separately authorized migration action.

## Capabilities

### New Capabilities

- `skillguard-author-maintenance-currentness`: Defines the current author identity, four-member unit boundary, target-owned validation inventory, clean consumer projection, and evidence/claim boundaries for SkillGuard 0.4 maintenance.

### Modified Capabilities

None. This change does not alter ResearchGuard, LogicGuard, SourceGuard, or TraceGuard domain behavior.

## Impact

- Author-only surfaces: `AGENTS.md`, `.skillguard/author-project.json`, and each maintained skill's `.skillguard` contract trio.
- FlowGuard process/structure/test models and focused verification records for this maintenance change.
- Repository-local OpenSpec artifacts and validation evidence.
- No global skill installation, package publication, Git commit/push/tag, or modification of the old standalone repositories.
